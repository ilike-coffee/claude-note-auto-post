# Pythonがなくても大丈夫？Claude APIの最初の一歩

「API」って聞くと、なんか難しそう……って感じませんか？

私も最初はそうでした。「エンジニアが使うもの」「プログラミングができないと無理」って思い込んでいて、ずっと避けていたんです。

でも実際に触ってみたら、**最初の一歩はびっくりするくらい簡単**でした。Pythonの知識がゼロに近い私でも、ちゃんと動かせたんです。

今日は「APIって聞いたことあるけど、自分には無理かな……」と思っているあなたに向けて、Claude APIの最初の一歩を一緒に踏み出してみましょう。

---

## そもそも「API」って何なの？

APIとは、**アプリケーション同士が話し合うための窓口**のことです。

ちょっとイメージしてみてください。レストランに行ったとき、あなたは直接キッチンに入ってシェフに料理してもらうわけじゃないですよね。ウェイターさんを通じて注文を伝えて、料理が出てきますよね。

APIはこのウェイターさんみたいな存在です。

Claude APIの場合、こんな流れになります。

1. あなたのプログラム（またはツール）が「このテキストを要約して」とAPIにリクエストを送る
2. AnthropicのサーバーにあるClaudeがそれを受け取って処理する
3. 回答がAPIを通してあなたのもとに返ってくる

ブラウザでclaude.aiを開いて話しかけるのと**やっていることは同じ**です。ただ、直接あなたの指が動かすのではなく、プログラムが代わりに動かすイメージです。

---

## Pythonって絶対必要なの？

「API = Python」みたいなイメージがありますよね。確かにPythonはよく使われます。

でも実は、**Pythonがなくても試せる方法があります**。

### 方法1：curlコマンドで試す

Macを使っている方なら、ターミナルというアプリが最初から入っています（Windowsの方はコマンドプロンプトかPowerShell）。

そこに以下のようなコマンドを貼り付けると、APIを呼び出せます。

```bash
curl https://api.anthropic.com/v1/messages \
  -H "x-api-key: あなたのAPIキー" \
  -H "anthropic-version: 2023-06-01" \
  -H "content-type: application/json" \
  -d '{
    "model": "claude-haiku-4-5-20251001",
    "max_tokens": 256,
    "messages": [{"role": "user", "content": "こんにちは！"}]
  }'
```

これを実行すると、Claudeから「こんにちは！どのようにお役に立てますか？」みたいな返答がJSON形式で返ってきます。

これだけです。Pythonは一切使っていません。

### 方法2：Anthropic Consoleのプレイグラウンドで試す

昨日ご紹介したAnthropicのConsole（管理画面）には「**Workbench**」というAPIのテスト画面があります。

ここではブラウザ上でAPIのリクエストを組み立てて、実際にClaudeに送ることができます。コードを書く必要すらありません。

「まずは本当に動くのか確認したい」という方には、このプレイグラウンドから試してみるのがおすすめです。

## 実際にやってみた

私も最初はcurlで試してみました。

ターミナルを開いて（Macのスポットライト検索で「ターミナル」と入力するだけで出てきます）、コマンドをコピーして、APIキーの部分だけ自分のものに書き換えて、エンターキーを押す。

すると……

```json
{
  "content": [
    {
      "text": "こんにちは！何かお手伝いできることはありますか？",
      "type": "text"
    }
  ],
  ...
}
```

こんな感じで返事が来ました。

**わあ、動いた！**

正直、この瞬間の感動は忘れられません。「私がコマンドを打ったら、AIが返事してくれた」という体験は、なんというかプログラミングの入口に立てた感じがしてとても嬉しかったです。

---

## Pythonで書くとどう変わるの？

curlで動くのはわかった。でも「なぜPythonを使う人が多いのか」も気になりますよね。

curlは1回だけ試すには便利ですが、**繰り返し使ったり、結果を加工したりするのが難しい**んです。

たとえば、「100件のメールを全部Claudeに要約させたい」となったとき、curlだと100回コマンドを打つことになってしまいます。

Pythonだとこれが数行で書けます。

```python
import anthropic

client = anthropic.Anthropic(api_key="あなたのAPIキー")

message = client.messages.create(
    model="claude-haiku-4-5-20251001",
    max_tokens=256,
    messages=[{"role": "user", "content": "こんにちは！"}]
)

print(message.content[0].text)
```

コードを見て「うわ、わからない……」と思った方、安心してください。私も最初はそうでした。

でもこれ、実は**読み方さえ知れば理解できる**んです。

- `anthropic.Anthropic()`でAnthropicとつながる準備をしている
- `client.messages.create()`で「メッセージを作って送って」と指示している
- `print()`で受け取った返答を画面に表示している

それだけです。

---

## 「model」の部分、何を選べばいいの？

上のコードに`model="claude-haiku-4-5-20251001"`と書きましたね。これはClaudeのどのバージョンを使うかを指定しています。

現在使えるモデルはいくつかあります。

| モデル名 | 特徴 |
|---|---|
| claude-haiku-4-5 | 軽量・高速・低コスト。シンプルなタスク向け |
| claude-sonnet-4-6 | バランス型。多くの用途に対応 |
| claude-opus-4-8 | 最高性能。複雑なタスク向け |

最初に試すなら、費用が安くて速い**claude-haiku-4-5**がおすすめです。慣れてきたら用途に合わせて使い分けてみましょう。

---

## 「max_tokens」って何？

もう一つ気になった方もいるかもしれません。`max_tokens`という項目です。

これは、**Claudeが返してくれる文章の長さの上限**を指定するものです。

「トークン」という単位は日本語だと1文字＝約1〜2トークンくらいです。256トークンなら大体100〜200文字くらいの返答が来ます。

長い文章を生成させたいときは1000や2000に増やせばOKです。ただし、トークンが増えると費用も上がるので、最初は少なめから試してみましょう。

## 今日やってみてほしいこと

APIを「難しそう」と遠ざけていた気持ち、少し変わってきましたか？

今日の「やってみてほしいこと」はひとつだけ。

**AnthropicのConsoleを開いて、WorkbenchでClaudeに何か話しかけてみる。**

コードを書かなくてもOK。ブラウザだけで体験できます。「おはよう」の一言だけでも、ちゃんと返事をしてくれます。

その体験が「あ、私でもAPIを使えるかも」という自信につながります。

## まとめ

- APIはプログラム同士が話し合うための窓口
- curlというコマンドを使えばPythonなしでも試せる
- AnthropicのConsole（Workbench）ならコードなしでブラウザから体験できる
- まずはHaikuモデルで気軽に試してみよう
