# Claude Code って何が便利なの？ターミナル版Claudeを触ってみた

「ターミナル」って聞いて、ちょっと身構えませんでしたか？

私も最初は「ターミナル」という言葉を見ただけで「あ、これはエンジニア向けだ。私には関係ない」と思っていました。

でも実際に触ってみると、**意外と面白くて、普通の人にも使いどころがある**ツールだとわかりました。今日は初心者目線でClaude Codeを触ってみたレポートをお届けします。

---

## そもそも「Claude Code」って何？

Claude CodeはAnthropicが提供している**CLIツール**です。

「CLI」とは「Command Line Interface」の略で、**テキストでコマンドを打ち込んで操作するツール**のこと。画面上のボタンを押すのではなく、キーボードで命令を文字入力して動かす、昔ながらのスタイルです。

ブラウザでclaude.aiを開いてチャットするのが「カフェでコーヒーを注文する」とすれば、Claude Codeは「自分のキッチンで直接コーヒーを淹れる」イメージ。手間は少し増えるけれど、自分のペースでより細かく動かせる、という感じです。

---

## どうやってインストールするの？

Claude Codeのインストールは、実はそんなに難しくありません。

まず必要なのは**Node.js**というソフト。JavaScriptを動かすための土台のようなものです。公式サイトからダウンロードしてインストールするだけでOK。

その後、ターミナルを開いて以下のコマンドを1行打ちます：

```bash
npm install -g @anthropic-ai/claude-code
```

「npm」というのはNode.jsの部品を管理するツール。上のコマンドは「claude-codeというパッケージをインストールして」という意味です。

インストール後、最初に一度だけAnthropicアカウントでログインする手順があります。ブラウザが自動で開いて認証できるので、ここも難しくないです。

## 実際に使ってみた感想

インストール後、ターミナルで`claude`とだけ打ってエンターを押すと……対話モードが始まります。

「こんにちは！何かお手伝いできることはありますか？」

なんだ、普通にチャットできるじゃないか（笑）。

でもClaude Codeが本当に便利なのは、**今いるフォルダのファイルをそのまま読んで処理してくれる**ところです。

たとえば、自分が書いた記事のテキストファイルがあって、そのファイルを指定しながら「この文章を要約して」と言うだけで、Claudeがファイルの中身を読んで要約してくれます。

いちいちファイルをコピーしてclaude.aiに貼り付けなくてもいい！これが地味に嬉しいポイントでした。

---

## Claude Codeの便利なところ3つ

実際に触ってみて「これは便利だな」と感じた点を3つ挙げます。

**1. ファイルを直接扱える**  
ローカル（自分のパソコン）にあるファイルをそのまま処理できます。ファイルのコピペ作業がなくなるだけで、作業効率がぐっと上がります。

**2. 繰り返し作業を自動化できる**  
「この形式で10件のメール下書きを作って」のような繰り返し作業を、一度のプロンプトでまとめてお願いできます。

**3. コード作成が完結する**  
「このスクリプトを作って動かして」といった作業がターミナル上で完結します。プログラミング経験が少ない私でも、手順を追えばなんとかなりました。

---

## 正直なところ、初心者には少し敷居が高い

メリットばかり書きましたが、正直に言うと**ターミナルに慣れていない人には少し敷居があります**。

コマンドを打ち間違えるとエラーが出る、エラーメッセージが英語でわからない……という壁があります。

ただ、面白い解決策があって——「エラーが出たらそのエラーメッセージをClaude Codeに貼り付けて『これどういう意味？』と聞く」という技が使えます（笑）。自分が自分の問題を解決してくれる、という少し不思議な体験でした。

## まとめ

- Claude CodeはターミナルからClaudeを使えるCLIツール
- Node.jsをインストール後、1つのコマンドで導入できる
- ローカルのファイルを直接扱える・繰り返し作業が得意
- 初心者にはとっつきにくいが、慣れると手放せなくなる

次回は、このClaude Codeを使って**実際に何かを自動化する体験**をレポートします。「Routines」という定期実行機能を使って、どんなことができるのか試してみます。お楽しみに！
