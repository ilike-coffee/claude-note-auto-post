# Claude学習note記事シリーズ 続編（day31〜）設計

## 背景

「Claude学習note記事の自動生成」ルーチン（Claude.ai Routines、GitHubリポジトリ
`ilike-coffee/claude-note-auto-post`）で初級編（day01〜day30）を投稿完了。
ルーチンは現在一時停止中。続編（中級編・上級編）を設計し、ある程度のところで
記事の有料化（マネタイズ）を始める。

## スコープ

**やること**
- 中級編（day31〜50）・上級編（day51〜）のコンテンツ構成設計
- `topics.yaml` へのトピック追記とデータ構造拡張（tier / is_paid / price）
- サムネイルの難易度別配色対応（`generate_thumbnail.py` 改修）
- day31〜33の不整合クリーンアップ（記事本文は削除済み、サムネイルのみ残存していた）

**やらないこと（別スコープ）**
- Noteへの自動投稿（下書き作成・公開・価格設定の自動化）— 認証情報の扱いや
  note規約との整合性など別途検討が必要な規模のため、今回は対象外。
  記事・サムネイルの生成まで自動化し、Noteへの反映は引き続き手動。
- ルーチン指示文・ローカル`.git/config`に平文で入っているGitHub PATの是正
  — セキュリティ上望ましくないが、今回のコンテンツ拡張とは独立した対応な
  ので別途扱う。

## 現状確認で判明した事実

- `articles/` は day01〜day30 のみ（ユーザー認識と一致）
- `thumbnails/` は day01〜day33 まで存在（day31〜33 は記事本文のみ削除された
  形跡があり、サムネイルだけ残っていた）
- `topics.yaml` の `current_day` は `34`
- → 本設計では `current_day` を `31` に戻し、day31から作り直す。
  `thumbnails/day31.png`〜`day33.png` は削除する。

## コンテンツ構成

### 全体像

| 区分 | day範囲 | 日数 | 公開 | サムネイル背景 |
|---|---|---|---|---|
| 初級編 | day01〜30 | 30日（済） | 無料 | オレンジ系（既存） |
| 中級編 | day31〜50 | 20日 | 無料 | 青系 |
| 上級編 | day51〜 | 未定（ブロック制で継続） | 有料 | 紫/深紺系 |

初級→中級は無料のまま連続させ、読者数と信頼を積み上げる。上級編から丸ごと
有料記事化し、無料/有料の境界を「day50までは全部無料、day51から先は有料」
とシンプルに保つ。

### 中級編（day31〜50、無料）

初級で基本を学んだ読者に、実践的な活用シーンと一歩踏み込んだ機能を伝える。

1. day31〜34: プロジェクト機能を使いこなす（Projects作成・ナレッジベース化・共有）
2. day35〜38: 自動化の入り口（カスタム指示・テンプレプロンプト・繰り返しタスクの効率化）
3. day39〜42: 検索・拡張機能（Web検索、Googleカレンダー/Gmail等のコネクタ活用）
4. day43〜46: ビジネス文書・データ活用実践（資料作成・簡易データ分析・表計算連携）
5. day47〜50: 中級のまとめ＋上級への橋渡し（Claude Code本格入門、上級への導線）

### 上級編（day51〜、有料）

中級で実践力をつけた読者に、開発・自動化・収益化という有料に見合う内容を
提供する。日数を固定せず、ブロック単位でネタが尽きるまで継続する。

1. day51〜55: Claude Codeで小さく開発する（ツール開発の第一歩、サブエージェント・スキル活用）
2. day56〜60: 自動化パイプラインを組む（本シリーズの自動生成システムのような自作事例、スケジュールタスク設計）
3. day61〜65: MCP・外部連携を作る側になる（自作MCPサーバー、独自コネクタ構築）
4. day66〜70: 収益化・仕事での本格導入（副業／業務でのClaude活用実例、チーム展開）
5. day71〜: ストック（プロンプトエンジニアリング上級、大規模プロジェクトのコンテキスト管理等）— 種切れ防止用に後で拡充

日々の個別トピック（title / keywords / tone / length / summary）は
`topics.yaml` に追記する形で用意する（本設計のレビュー後、実装時に作成）。

### 価格

上級編は1記事 300〜500円のレンジで、まずは一律 **380円** を初期値とする
（分量や内容の濃さに応じて個別に調整可能な形にしておく）。

## サムネイル設計

`generate_thumbnail.py` は1280×670pxのPillow生成で、左にロボット＋
「Claude素人」の文字、右にDay番号とタイトル、という構図は変更しない
（ロボットの形状・フォントは固定）。変えるのは背景色のみ。

### 配色（試作して確認済み）

| tier | メイン | アクセント | ダーク（装飾円・目/口） |
|---|---|---|---|
| 初級 | `#E8630A` | `#F5A623` | `#C44D00`（既存のまま） |
| 中級 | `#1E5FA8` | `#4FA8D8` | `#123F73` |
| 上級 | `#4B2E83` | `#7A5AC2` | `#2E1B54` |

現状 `_draw_robot()` 内でロボットの目・口の色が `"#E8630A"` に固定
ハードコードされている。これを各tierのメインカラーに連動させる修正が
必要（そうしないと中級・上級でも目・口だけオレンジのままになる）。

### 実装方式

`generate_thumbnail.py` が `topics.yaml` を読み込み、渡された `day` に
対応する `tier` を自動判定してパレットを選択する。ルーチンの実行コマンド
自体（`generate_thumbnail.py <day> "<title>" <output_path>`）は変更しない。

## データ構造変更（topics.yaml）

各トピックに `tier` / `is_paid` / `price` を追加する。

```yaml
current_day: 31

topics:
  - day: 31
    tier: "中級"
    title: "..."
    keywords: [...]
    tone: "..."
    length: "..."
    summary: "..."
    is_paid: false
    price: null

  - day: 51
    tier: "上級"
    title: "..."
    keywords: [...]
    tone: "..."
    length: "..."
    summary: "..."
    is_paid: true
    price: 380
```

`is_paid` / `price` は将来Note自動投稿を実装する際に使う想定のメタデータ
であり、今回のスコープでは記事生成・サムネイル生成の挙動そのものには
影響しない（Noteでの実際の価格設定は引き続き手動）。

## ルーチンの指示文への反映

現行の「Claude学習note記事の自動生成」ルーチンの指示は、`topics.yaml` の
`current_day` を読んでトピックを参照する構造なので、`topics.yaml` に
day31以降のエントリを追記するだけでルーチン本体の指示文は大きく変える
必要がない。ただし以下を追記する：

- 中級編・上級編それぞれの執筆トーンの補足（初級より一歩踏み込んだ実践
  トーンにする、上級は開発・自動化寄りの内容にする、等）
- サムネイル生成コマンドは変更なし（tier判定はスクリプト内部で完結）
